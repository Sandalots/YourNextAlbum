# Output CSV Files Overview

This document describes the three main CSV files generated by the Album Recommendation System in the `outputs/` directory:

---

## 1. `pitchfork_reviews_raw.csv`
**Description:**
- Raw scraped reviews from Pitchfork.
- Contains all original review text, scores, album/artist info, and URLs as collected by the scraper.
- No preprocessing or feature engineering applied.

**Columns:**
- `artist_name`, `album_name`, `review_text`, `score`, `release_year`, `genre`, `review_url`, `album_art_url`, etc.

---

## 2. `pitchfork_reviews_preprocessed.csv`
**Description:**
- Cleaned and preprocessed version of the raw reviews.
- Text is tokenized, lemmatized, and cleaned using NLTK.
- Missing values filled, basic features engineered.

**Columns:**
- All columns from the raw file, plus:
- `review_text_processed`, `score_normalized`, `unique_word_ratio`, etc.

---

## 3. `pitchfork_reviews_preprocessed_plus_sentiments.csv`
**Description:**
- Fully analyzed dataset with all extracted features and sentiment results.
- Includes columns for sentiment, mood, instrumentation, production, novelty, context, and more, generated by the sentiment analysis and feature extraction pipeline.

**Columns:**
- All columns from the preprocessed file, plus:
- `sentiment_label`, `sentiment_score`, `themes`, `key_highlights`, `lyrical_themes`, `musical_characteristics`, `instrumentation`, `production_quality`, `mood_energy`, `is_polarizing`, `novelty_score`, `temporal_context`, `listening_contexts`, `critical_consensus`, etc.

---

For more details on how these files are generated, see the main README.md.
